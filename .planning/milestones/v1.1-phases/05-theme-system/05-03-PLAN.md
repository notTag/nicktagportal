**Inert attribute management:**
When mobile menu opens, the main content behind it must be inert for accessibility (per UI-SPEC and D-06 focus trap).

Add a watcher:

```typescript
watch(isMobileMenuOpen, (open) => {
  const appContent = document.getElementById('app-content')
  if (appContent) {
    if (open) {
      appContent.setAttribute('inert', '')
      appContent.setAttribute('aria-hidden', 'true')
    } else {
      appContent.removeAttribute('inert')
      appContent.removeAttribute('aria-hidden')
    }
  }
})
```

**Template changes:**

The outer `<div>` that wraps header + main + footer gets `id="app-content"` so the inert watcher can target it.

MobileMenu is rendered OUTSIDE the app-content div (sibling, not child) so it is NOT inert when open.
