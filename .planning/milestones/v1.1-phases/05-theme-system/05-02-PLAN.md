Create `packages/ui/src/components/ThemeDropdown.vue` as a Vue 3 SFC with `<script setup lang="ts">`. This component is the WAI-ARIA listbox theme picker per D-01, D-02, D-04, D-05, D-06.

**Component behavior:**

The component imports `useThemeStore` from `@/stores/theme` and `themeList` from `@/themes`. It manages local state for: `isOpen` (boolean), `activeIndex` (number, -1 when closed).

**Trigger button:**

- Text content: `Theme &#x25BE;` (the word "Theme" + Unicode small down-pointing triangle U+25BE)
- Classes: `text-sm font-normal text-text-muted hover:text-accent cursor-pointer transition-colors`
- ARIA: `role="button"`, `aria-haspopup="listbox"`, `aria-expanded` bound to isOpen
- Click handler: toggles dropdown open/closed. When opening, sets activeIndex to the index of the current confirmed theme in themeList.
- Keyboard: Enter, Space, or ArrowDown opens dropdown (if closed)

**Dropdown panel:**

- Rendered conditionally when `isOpen` is true (v-if)
- Position: `absolute right-0 top-full mt-2` (below trigger, right-aligned)
- Styles: `bg-surface-overlay border border-border rounded-lg py-2 w-[220px]` plus `shadow-[0_4px_16px_rgba(0,0,0,0.3)]`
- z-index: `z-40`
- ARIA: `role="listbox"`, `aria-label="Select theme"`, `aria-activedescendant` pointing to the ID of the active option

**Option rows:**

- One `<div>` per theme in `themeList`
- Each option: `role="option"`, `aria-selected` true if this is the confirmed theme (store.confirmedThemeId), unique `id` of format `theme-option-{theme.id}`
- Classes: `px-2 py-1.5 text-sm text-text cursor-pointer rounded mx-1`
- Conditional classes: `bg-selection` when index equals activeIndex (keyboard focused), `hover:bg-hover` for mouse hover
- Content: checkmark prefix (`\u2713 ` in `text-accent` span) shown only when theme.id === store.confirmedThemeId, then theme.name
- Click handler: `store.setTheme(theme.id)` then close dropdown
- Mouse enter: set activeIndex to this item's index AND call `store.previewTheme(theme.id)` for live preview on hover

**Keyboard interaction (on the wrapper div, using @keydown):**

- ArrowDown: prevent default, increment activeIndex (clamp to list length - 1), call `store.previewTheme(themeList[activeIndex].id)`
- ArrowUp: prevent default, decrement activeIndex (clamp to 0), call `store.previewTheme(themeList[activeIndex].id)`
- Enter: prevent default, call `store.setTheme(themeList[activeIndex].id)`, close dropdown
- Escape: prevent default, call `store.revertPreview()`, close dropdown
- Tab: call `store.revertPreview()`, close dropdown (do not prevent default -- let focus move naturally)

**Click-outside handling:**

- Use a ref on the wrapper div. On mounted, add a document click listener that checks if the click target is outside the wrapper. If outside: call `store.revertPreview()`, close dropdown. On unmounted, remove the listener.

**Open/close logic:**

- `open()`: set isOpen=true, set activeIndex to index of confirmedThemeId in themeList, use nextTick to focus the listbox panel
- `close()`: set isOpen=false, set activeIndex=-1

Add `ThemeDropdown` to the barrel export in `packages/ui/src/index.ts`.

- `showThemePicker: boolean` (default false) -- controls ThemeDropdown visibility per D-07 feature flag
- `isMobileMenuOpen: boolean` (default false) -- controls hamburger icon state (lines vs X)

**Emits to add:**

- `toggle-menu` -- emitted when hamburger button is clicked (MobileMenu state managed by parent)

**Desktop layout (>= sm / 640px):**

- Existing name span and nav stay as-is
- Nav links get `hidden sm:flex` to hide on mobile
- Add ThemeDropdown AFTER nav, pushed right with `ml-auto`. Wrap in `v-if="showThemePicker"`. Add `hidden sm:block` to hide on mobile.

**Mobile layout (< sm / 640px):**

- Nav links: hidden (already handled by `hidden sm:flex`)
- ThemeDropdown: hidden (already handled by `hidden sm:block`)
- Hamburger button: visible. Add `sm:hidden` so it only shows on mobile.
- Hamburger pushed right with `ml-auto`

**Hamburger button markup:**

- `<button>` with `class="sm:hidden ml-auto flex h-11 w-11 flex-col items-center justify-center gap-1.5"` (44px touch target per WCAG 2.5.5)
- `aria-label="Toggle menu"`, `aria-expanded` bound to `isMobileMenuOpen`
- Three `<span>` elements for the three lines: each `block h-0.5 w-6 bg-text transition-transform duration-200`
- When `isMobileMenuOpen` is true, animate to X:
  - Top bar: `translate-y-2 rotate-45`
  - Middle bar: `opacity-0`
  - Bottom bar: `-translate-y-2 -rotate-45`
- Click handler: `$emit('toggle-menu')`

**Import changes:**

- Add `import ThemeDropdown from './ThemeDropdown.vue'`
- Add `defineProps` and `defineEmits` with proper types
