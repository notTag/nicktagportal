**1. `apps/shell/src/terminal/vfs/filesystem.ts`** -- Core VFS operations:

Export these functions:

- `normalizePath(path: string): string` -- Split on `/`, filter empty, resolve `.` (skip) and `..` (pop), return `'/' + resolved.join('/')`. Root is `/home/nick` which maps to `~`.
- `resolvePath(cwd: string, input: string): string` -- If input starts with `~`, replace with `/home/nick`. If starts with `/`, normalize directly. Otherwise, normalize `cwd + '/' + input`.
- `getNode(root: VfsDirectory, absolutePath: string): VfsNode | null` -- Walk the tree from root following path segments. Return null if any segment not found.
- `getParentAndName(root: VfsDirectory, absolutePath: string): { parent: VfsDirectory; name: string } | null` -- Get parent directory and final path segment. Used by mkdir/touch.
- `createFilesystem(): VfsDirectory` -- Create root `/` directory, add `home/nick` directory chain, call `buildResumeTree()` to populate `~/` children, then call `loadUserOverlay()` and merge user-created nodes into the tree.

The home path constant: `export const HOME_DIR = '/home/nick'`

Create 3 command handler modules. Every handler has signature `(args: string[], ctx: CommandContext) => void` and writes output via `ctx.terminal.writeln()`.

**1. `apps/shell/src/terminal/commands/navigation.ts`** -- Export: `cdCommand`, `lsCommand`, `pwdCommand`, `treeCommand`

`cdCommand` (CLI-03):

- No args: cd to `~` (HOME_DIR)
- Resolve path via `resolvePath(ctx.cwd, args[0])`
- Get node via `getNode(ctx.root, resolvedPath)`
- If null: `ctx.terminal.writeln(ansi.red('cd: no such file or directory: ' + args[0]))`
- If file: `ctx.terminal.writeln(ansi.red('cd: not a directory: ' + args[0]))`
- If directory: `ctx.setCwd(resolvedPath)`

`lsCommand` (CLI-04):

- Target path from args[0] or ctx.cwd
- Get node, validate is directory
- Iterate children Map, write each name. Directories get trailing `/` and ansi.cyan coloring. Files in default color.
- If path not found: `ansi.red('ls: cannot access \'' + path + '\': No such file or directory')`

`pwdCommand` (CLI-06):

- `ctx.terminal.writeln(ctx.cwd)`

`treeCommand` (CLI-11):

- Target path from args[0] or ctx.cwd
- Recursive traversal with box-drawing characters: `├──` for non-last, `└──` for last, `│   ` for continuation, `    ` for spacing
- Directories colored with ansi.cyan, files in default
- Show directory count and file count at bottom: `\nN directories, M files`

**2. `apps/shell/src/terminal/commands/files.ts`** -- Export: `catCommand`, `touchCommand`, `mkdirCommand`

`catCommand` (CLI-05, VFS-02):

- Resolve path, get node
- If null: `ansi.red('cat: ' + path + ': No such file or directory')`
- If directory: `ansi.red('cat: ' + path + ': Is a directory')`
- If file: `ctx.terminal.writeln(node.content)` -- content already has ANSI codes from resumeData.ts

`touchCommand` (CLI-16, VFS-03):

- Resolve path, check if already exists (no-op if file exists)
- Get parent directory via `getParentAndName`
- If parent doesn't exist: error
- Create `VfsFile` with `{ type: 'file', name, content: '', readonly: false }`
- Add to parent.children
- Save to localStorage overlay via `saveUserOverlay`

`mkdirCommand` (CLI-15, VFS-03):

- Resolve path, check if exists: `ansi.red("mkdir: cannot create directory '" + name + "': File exists")`
- Get parent directory
- Create `VfsDirectory` with `{ type: 'directory', name, children: new Map(), readonly: false }`
- Add to parent.children
- Save to localStorage overlay
