2. Create `.github/workflows/deploy.yml` with this exact content:

```yaml
# Deploy shell app to GitHub Pages on push to main
# TODO: Replace with AWS deployment (CloudFront + S3) when micro-frontend remotes are introduced
name: Deploy to GitHub Pages

on:
  push:
    branches:
      - main

concurrency:
  group: pages-${{ github.ref }}
  cancel-in-progress: true

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: oven-sh/setup-bun@v2
        with:
          bun-version: '1.1'

      - name: Install dependencies
        run: bun install --frozen-lockfile

      - name: Build shell app
        working-directory: apps/shell
        run: bun run build

      - name: Create 404.html for SPA routing
        run: cp apps/shell/dist/index.html apps/shell/dist/404.html

      - name: Upload Pages artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: apps/shell/dist

  deploy:
    needs: build
    runs-on: ubuntu-latest
    permissions:
      pages: write
      id-token: write
    environment:
      name: production
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
```

Key decisions implemented:

- Per D-01: `cp index.html 404.html` after build (not a committed file per D-08)
- Per D-03: Two jobs — build runs automatically, deploy requires approval via `environment: production`
- Per D-04: `actions/upload-pages-artifact@v3` + `actions/deploy-pages@v4`
- Per D-05: AWS migration TODO comment at top of file (satisfies DEPLOY-04)
- Per D-06: `permissions: pages: write, id-token: write` scoped to deploy job only (anti-pattern avoidance)
- Bun version `1.1` (matches project constraint `^1.1` from CLAUDE.md; Claude's discretion on exact version)
- `bun install --frozen-lockfile` for reproducible CI installs (Claude's discretion)
- `concurrency` group prevents overlapping deploys
- `bun run build` in `working-directory: apps/shell` — install runs from repo root for workspace resolution (Pitfall 5 avoidance)

3. Create `.github/workflows/rollback.yml` with this exact content:

```yaml
# Roll back to a previous deployment by re-deploying its artifact
# Usage: Actions tab > Rollback Deployment > Run workflow > enter run_id from a prior deploy.yml run
name: Rollback Deployment

on:
  workflow_dispatch:
    inputs:
      run_id:
        description: 'Run ID of the deploy.yml run to roll back to (find in Actions tab URL)'
        required: true
        type: string

jobs:
  rollback:
    runs-on: ubuntu-latest
    permissions:
      pages: write
      id-token: write
      actions: read
    environment:
      name: production
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - name: Download artifact from previous run
        uses: actions/download-artifact@v4
        with:
          name: github-pages
          run-id: ${{ inputs.run_id }}
          github-token: ${{ secrets.GITHUB_TOKEN }}

      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
```

Key decisions implemented:

- Per D-03: `workflow_dispatch` only with `run_id` string input
- `actions: read` permission included (Pitfall 2 — required for cross-run artifact download)
- Same `environment: production` gate as deploy.yml
- `github-token` provided for cross-run artifact access
