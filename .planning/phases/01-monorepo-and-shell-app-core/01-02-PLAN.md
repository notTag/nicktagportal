```vue
<script setup lang="ts">
import { RouterLink } from 'vue-router'
</script>

<template>
  <header class="bg-surface-raised border-b border-border">
    <div class="max-w-5xl mx-auto w-full px-4 h-16 flex items-center justify-between">
      <span class="text-base font-bold text-text">nick-site</span>
      <nav class="flex items-center gap-2">
        <RouterLink
          to="/"
          class="text-sm font-normal text-text-muted hover:text-accent transition-colors"
          active-class="text-accent"
        >
          Home
        </RouterLink>
        <RouterLink
          to="/playground"
          class="text-sm font-normal text-text-muted hover:text-accent transition-colors"
          active-class="text-accent"
        >
          Playground
        </RouterLink>
      </nav>
    </div>
  </header>
</template>
```

Key details:
- Background: `bg-surface-raised` (slate-800, #1e293b)
- Height: `h-16` (64px per UI-SPEC)
- Inner content: `max-w-5xl mx-auto w-full px-4` (constrained width per UI-SPEC)
- Site name: "nick-site" left-aligned in `text-base font-bold text-text`
- Nav links: "Home" and "Playground" right-aligned as `<RouterLink>` elements
- Nav link style: `text-sm font-normal text-text-muted hover:text-accent transition-colors`
- Active link: `active-class="text-accent"` (uses Vue Router's active class prop)
- Bottom border: `border-b border-border`
- Nav gap: `gap-2` (8px per spacing scale)

NOTE: Use `active-class` prop on `<RouterLink>` instead of relying on `router-link-active` CSS class. For the Home link, you may need `exact-active-class` instead of `active-class` to prevent it from being active on all routes. Consider using:
- Home: `exact-active-class="!text-accent"` (exact match for `/` only)
- Playground: `active-class="!text-accent"` (matches `/playground` and `/playground/:remote`)

The `!` prefix ensures the active class overrides the default `text-text-muted`. Alternatively, structure the classes so active state wins without `!important`.
