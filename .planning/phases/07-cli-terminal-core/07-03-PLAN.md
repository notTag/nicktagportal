**1. `apps/shell/src/terminal/commands/terminal.ts`** -- Export: `clearCommand`, `aliasCommand`

`clearCommand` (CLI-09):

- `ctx.terminal.clear()` -- clears scrollback and viewport
- Prompt rewrite handled by Shell after command returns

`aliasCommand` (CLI-17):

- No args: list all aliases. For each in `ctx.aliases`: `alias name='command'`
- With arg in format `name=command` or `name='command'`: parse name and command, call `ctx.setAlias(name, command)`
- Invalid format: `ansi.red('alias: usage: alias name=command')`

Note: `CLITHEME=<name>` is NOT a command -- it's a variable assignment handled by the Shell class before command dispatch. When Shell sees input matching `/^CLITHEME=(.+)$/i`, it calls `ctx.setTheme(value)` directly without going through the command registry. This is per D-06.

Create `apps/shell/src/terminal/Shell.ts` -- the central orchestrator class.

```typescript
export class Shell {
  private terminal: Terminal
  private lineBuffer: string
  private cursorPos: number
  private cwd: string
  private root: VfsDirectory
  private history: string[]
  private historyIndex: number
  private aliases: Record<string, string>
  private terminalThemeStore: ReturnType<typeof useTerminalThemeStore>

  constructor(
    terminal: Terminal,
    terminalThemeStore: ReturnType<typeof useTerminalThemeStore>,
  )
  init(): void // Write banner, write first prompt, wire onData
  dispose(): void // Cleanup
}
```

**Constructor:**

- Store terminal reference and theme store
- Initialize: `lineBuffer = ''`, `cursorPos = 0`, `cwd = HOME_DIR` (which is `/home/nick`)
- Call `createFilesystem()` to build VFS root
- Load history from localStorage key `nicksite-cli-history` (default `[]`, max 100 entries)
- Load aliases from localStorage key `nicksite-cli-aliases` (default `{}`)
- `historyIndex = -1` (not navigating history)

**init():**

- Write banner via `terminal.write(generateBanner())`
- Write initial prompt via `writePrompt()`
- Wire `terminal.onData(this.handleData.bind(this))`

**writePrompt():**

- Per D-14 colored prompt: `nick` (ANSI green \x1b[32m), `@` (ANSI bright black \x1b[90m), `nicktag.tech` (ANSI cyan \x1b[36m), `:` (white), `displayPath` (ANSI yellow \x1b[33m), `$` (white \x1b[37m)
- `displayPath`: replace `/home/nick` prefix with `~` in cwd. If cwd IS `/home/nick`, display `~`.
- Write via `terminal.write(promptString)` (no newline -- user types on same line)

**handleData(data: string):**

- `\r` (Enter):
  - `terminal.writeln('')` (newline after user input)
  - Trim lineBuffer. If non-empty:
    - Add to history array, save to localStorage `nicksite-cli-history` (keep last 100)
    - Call `executeCommand(lineBuffer.trim())`
  - Reset lineBuffer, cursorPos, historyIndex
  - Call `writePrompt()`

- `\x7f` (Backspace):
  - If cursorPos > 0: remove char at cursorPos-1 from lineBuffer, decrement cursorPos
  - Write `\b \b` to erase visual character (end-of-line editing only)

- `\x1b[A` (Up arrow -- CLI-12):
  - If history not empty and can go further back: increment historyIndex (from current position toward 0)
  - Clear current line visually: write `\r` + prompt + spaces to clear + `\r` + prompt
  - Set lineBuffer to `history[history.length - 1 - historyIndex]`
  - Write lineBuffer to terminal
  - Update cursorPos

- `\x1b[B` (Down arrow -- CLI-12):
  - Decrement historyIndex. If below 0, clear line (back to empty input)
  - Otherwise show history entry
  - Same visual clear/rewrite pattern

- `\t` (Tab -- CLI-13):
  - Parse lineBuffer to determine if completing a command name or a file path
  - If lineBuffer has no space: completing command name. Match prefix against `getCommandNames()`
  - If lineBuffer has space: completing file path. Get last token, resolve partial path against cwd, match against VFS directory children names
  - If exactly one match: complete it (write remaining chars to terminal and append to lineBuffer). For directories, append `/`
  - If multiple matches: write all matches on new line, then rewrite prompt + lineBuffer
  - If no matches: do nothing (bell optional)

- Default (printable chars, `data >= ' '`):
  - Append to lineBuffer at cursorPos, increment cursorPos
  - Write data to terminal

**executeCommand(input: string):**

- Check for alias expansion: split input on first space, if first word is in aliases, replace with alias value
- Check for CLITHEME assignment per D-06: if input matches `/^CLITHEME=(.+)$/i`:
  - Extract theme name, normalize (lowercase, replace spaces with hyphens)
  - Call `terminalThemeStore.setTheme(themeId)` if valid
  - Apply new theme to terminal: `terminal.options.theme = { ...terminalThemeStore.xtermTheme }` (per research Pitfall 5: must spread)
  - Write confirmation: `Terminal theme set to: {themeName}`
  - If invalid theme name: `ansi.red('Unknown theme: ' + name + '. Available: ' + themeList)` listing all 9 theme IDs
  - Return (don't dispatch to command registry)
- Split input into command name + args (split on whitespace, first token = command)
- Look up in `commandRegistry`
- If found: call handler with args and a `CommandContext` object built from Shell state
- If not found (CLI-14): `terminal.writeln(ansi.red(commandName + ': command not found'))`

**CommandContext construction:**

- `terminal`: this.terminal
- `cwd`: this.cwd
- `setCwd`: method that updates this.cwd
- `root`: this.root
- `history`: this.history (copy for reading)
- `aliases`: this.aliases
- `setAlias`: method that updates this.aliases and saves to localStorage `nicksite-cli-aliases`
- `setTheme`: method that calls terminalThemeStore.setTheme and applies to terminal

**localStorage persistence:**

- History: key `nicksite-cli-history`, save on every command execution, max 100 entries
- Aliases: key `nicksite-cli-aliases`, save on every alias set
- Both use try/catch for quota safety
