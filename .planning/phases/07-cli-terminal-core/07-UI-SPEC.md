## Typography

This phase has two typography domains: the **page container** (Tailwind) and the **terminal interior** (xterm.js canvas renderer).

### Page Container (Tailwind -- outer wrapper only)

| Role | Size | Weight        | Line Height |
| ---- | ---- | ------------- | ----------- |
| Body | 14px | 400 (regular) | 1.5         |

Only one role needed -- the page container has no headings or display text. The terminal fills the view.

### Terminal Interior (xterm.js ITheme/options)

| Role             | Size                    | Weight        | Line Height |
| ---------------- | ----------------------- | ------------- | ----------- |
| Terminal text    | 14px                    | 400 (regular) | 1.2         |
| Banner ASCII art | 14px (same as terminal) | 400 (regular) | 1.2         |

xterm.js `fontSize: 14`, `lineHeight: 1.2`. All terminal text is monospace at a single size. Bold weight (700) used only by xterm.js SGR bold escape sequences for prompt segments and colored output headers.

Font weights declared: 400 (regular) + 700 (bold via ANSI escape). Two weights total.

Source: D-14 specifies colored prompt segments. CONTEXT.md Claude's Discretion includes terminal font size and line height.

## Color

This phase uses two independent color systems: the **site theme** (CSS custom properties from Phase 5) for the page background, and the **terminal theme** (xterm.js ITheme) for the terminal interior.

### Page Container (site theme -- inherits active site theme)

| Role            | Value                         | Usage                                |
| --------------- | ----------------------------- | ------------------------------------ |
| Dominant (60%)  | `var(--color-surface)`        | Page background behind terminal      |
| Secondary (30%) | `var(--color-surface-raised)` | Not used -- terminal fills the view  |
| Accent (10%)    | `var(--color-accent)`         | Not used at page level in this phase |

### Terminal Interior (xterm.js ITheme -- independent from site theme)

The terminal has its own color palette derived from the `$CLITHEME` variable (default: Dark Modern). Colors are mapped from the existing `ThemeColors` interface to xterm.js `ITheme` properties.

| xterm.js ITheme Property | Maps From ThemeColors | Dark Modern Value | Usage                              |
| ------------------------ | --------------------- | ----------------- | ---------------------------------- |
| `background`             | `surface`             | `#1F1F1F`         | Terminal background (60% dominant) |
| `foreground`             | `text`                | `#CCCCCC`         | Default text color                 |
| `cursor`                 | `accentYellow`        | `#CCA700`         | Blinking block cursor              |
| `cursorAccent`           | `surface`             | `#1F1F1F`         | Cursor text color (inverse)        |
| `selectionBackground`    | `selection`           | `#04395e`         | Text selection highlight           |
| `selectionForeground`    | `text`                | `#CCCCCC`         | Selected text color                |

### Terminal ANSI Color Mapping (for prompt segments and output)

Per D-14, the colored prompt uses these semantic assignments:

| Prompt Segment              | ANSI Color                   | Maps From ThemeColors | Dark Modern Value |
| --------------------------- | ---------------------------- | --------------------- | ----------------- |
| `nick`                      | Green (ANSI 2)               | `accentCyan`          | `#4daafc`         |
| `@`                         | Gray (ANSI 8 / bright black) | `textMuted`           | `#9D9D9D`         |
| `nicktag.tech`              | Cyan (ANSI 6)                | `accentCyan`          | `#4daafc`         |
| `~/path`                    | Yellow (ANSI 3)              | `accentYellow`        | `#CCA700`         |
| `$`                         | White (ANSI 7)               | `text`                | `#CCCCCC`         |
| Error text (CLI-14)         | Red (ANSI 1)                 | `destructive`         | `#F85149`         |
| Banner border               | Bright cyan                  | `accentCyan`          | `#4daafc`         |
| Banner title                | Bright white                 | `textOnAccent`        | `#FFFFFF`         |
| File content headers (D-18) | Accent                       | `accent`              | `#0078D4`         |

Accent reserved for: banner border/title coloring, prompt `nick` and `nicktag.tech` segments, file content section headers (Role, Period, Team, Highlights per D-18). Never used for body text or command output.

Destructive reserved for: unknown command error messages (CLI-14) only.

Source: D-05 (Dark Modern default), D-07 (9 curated themes), D-14 (colored prompt), D-15 (prompt adapts to CLITHEME), D-18 (colored file headers), CLI-14 (red error text).

## Copywriting Contract

| Element                        | Copy                                                                                                                |
| ------------------------------ | ------------------------------------------------------------------------------------------------------------------- |
| Primary CTA                    | "Type 'help' for commands, or try: ls ~/work" (post-banner hint, per D-13)                                          |
| Empty state heading            | Not applicable -- terminal always shows the welcome banner on load                                                  |
| Empty state body               | Not applicable -- the virtual filesystem is pre-populated (VFS-01)                                                  |
| Error state (unknown command)  | "{command}: command not found" (per CLI-14, rendered in red)                                                        |
| Error state (bad path)         | "cd: no such file or directory: {path}" / "cat: {path}: No such file or directory"                                  |
| Error state (cat on directory) | "cat: {path}: Is a directory"                                                                                       |
| Error state (cd to file)       | "cd: not a directory: {path}"                                                                                       |

| Help command output heading    | "Available commands:"                                                                                               |

| Welcome banner subtitle        | "interactive resume" (per D-12)                                                                                     |
| Welcome banner title           | "nicktag.tech" (per D-10, rendered in ASCII box art)                                                                |
| Destructive confirmation       | Not applicable -- no destructive actions in this phase. `mkdir` and `touch` create content; no `rm` command exists. |

Source: D-10 through D-13 (banner/prompt), D-08 (CLITHEME in help), CLI-07 (whoami), CLI-14 (error format). Error message formats follow standard Unix shell conventions.

## Component Inventory

This phase produces these visual elements (not traditional UI components -- the terminal is canvas-rendered by xterm.js):

| Element              | Type                    | Description                                                                |
| -------------------- | ----------------------- | -------------------------------------------------------------------------- |
| CliView.vue          | Vue view (rewrite)      | Full-height terminal container, replaces current placeholder               |
| Terminal container   | Tailwind-styled `<div>` | Wraps xterm.js, handles click-to-focus (D-03), responsive fit (D-04)       |
| Welcome banner       | ANSI escape art         | Box-drawing characters around "nicktag.tech" + subtitle (D-10, D-11, D-12) |
| Colored prompt       | ANSI escape sequence    | `nick@nicktag.tech:~/path $` with per-segment colors (D-14)                |
| File content display | ANSI-formatted text     | Structured headers with colored labels for cat output (D-18)               |

### Layout Contract

Per D-01 and D-02:

- Terminal renders **below the site header** -- `TheHeader` remains visible and interactive
- Terminal container uses the **same max-width as other site pages** (`max-w-5xl mx-auto px-4`)
- Terminal fills **remaining vertical space** below header: `calc(100vh - header-height - vertical-padding)`
- Container has `bg-surface` from site theme (provides contrast border around terminal)
- xterm.js canvas fills 100% of the container interior with its own `background` from `$CLITHEME`

### Interaction Contract

| Interaction                          | Behavior                                                                     |
| ------------------------------------ | ---------------------------------------------------------------------------- |
| Click anywhere in terminal container | Focus xterm.js for keyboard input (D-03)                                     |
| Window resize                        | xterm.js fit addon reflows columns/rows (D-04)                               |
| Type command + Enter                 | Execute command, display output, show new prompt                             |
| Up/Down arrow                        | Navigate command history (CLI-12)                                            |
| Tab key                              | Autocomplete commands and file paths (CLI-13)                                |
| `CLITHEME=<name>`                    | Immediately update all terminal colors, persist to localStorage (D-06, D-09) |

## Terminal Theme Mapping Reference

For executor use -- how each of the 9 site themes maps to xterm.js ITheme. The mapping function reads from the existing `ThemeColors` interface (already defined in `apps/shell/src/themes/types.ts`).

```
ThemeColors.surface       -> ITheme.background
ThemeColors.text          -> ITheme.foreground
ThemeColors.accentYellow  -> ITheme.cursor
ThemeColors.surface       -> ITheme.cursorAccent
ThemeColors.selection     -> ITheme.selectionBackground
ThemeColors.text          -> ITheme.selectionForeground
ThemeColors.destructive   -> ANSI red (color index 1)
ThemeColors.accentCyan    -> ANSI green (color index 2), ANSI cyan (color index 6)
ThemeColors.accentYellow  -> ANSI yellow (color index 3)
ThemeColors.textMuted     -> ANSI bright black (color index 8)
ThemeColors.text          -> ANSI white (color index 7)
ThemeColors.accent        -> ANSI blue (color index 4)
ThemeColors.link          -> ANSI magenta (color index 5)
ThemeColors.textOnAccent  -> ANSI bright white (color index 15)
```

Source: D-05, D-07, D-15. Claude's Discretion: "How $CLITHEME maps site theme tokens to xterm.js ITheme properties."
