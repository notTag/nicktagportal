## Color

This phase is unique: the color contract is **not a single palette** but a **16-variable CSS custom property system** that swaps entirely per theme. The contract defines the variable roles and the default values (SynthWave '84).

### Color Variable Contract (16 variables)

| Variable                  | Role                                   | Default (SynthWave '84) | 60/30/10 Role   |
| ------------------------- | -------------------------------------- | ----------------------- | --------------- |
| `--color-surface`         | Main page background                   | `#262335`               | Dominant (60%)  |
| `--color-surface-raised`  | Cards, header, footer, raised surfaces | `#241b2f`               | Secondary (30%) |
| `--color-surface-overlay` | Modals, dropdowns, overlays            | `#171520`               | Secondary (30%) |
| `--color-text`            | Primary text                           | `#ffffff`               | --              |
| `--color-text-muted`      | Secondary/description text             | `#848bbd`               | --              |
| `--color-text-on-accent`  | Text on accent-colored backgrounds     | `#ffffff`               | --              |
| `--color-accent`          | Primary interactive accent             | `#ff7edb`               | Accent (10%)    |
| `--color-accent-cyan`     | Secondary accent (decorative)          | `#72f1b8`               | Accent (10%)    |
| `--color-accent-yellow`   | Tertiary accent (decorative)           | `#fede5d`               | Accent (10%)    |
| `--color-destructive`     | Error states, destructive actions      | `#fe4450`               | Semantic        |
| `--color-link`            | Link text color                        | `#f97e72`               | Semantic        |
| `--color-link-hover`      | Link hover color                       | `#ff7edb`               | Semantic        |
| `--color-border`          | Borders, dividers                      | `#495495`               | Chrome          |
| `--color-header-bg`       | Header background                      | `#241b2f`               | Chrome          |
| `--color-selection`       | Selected/active item background        | `#ffffff20`             | State           |
| `--color-hover`           | Hover item background                  | `#37294d99`             | State           |

### 60/30/10 Split

- **60% Dominant:** `surface` -- all full-page backgrounds, main content areas
- **30% Secondary:** `surfaceRaised`, `surfaceOverlay`, `headerBg` -- header, footer, cards, dropdown panels, mobile menu
- **10% Accent:** `accent` reserved for: active nav link indicator, theme dropdown trigger hover, primary CTA button background, currently-selected theme checkmark, link text, focus rings

### Accent Reserved-For List (explicit)

`--color-accent` is used ONLY on:

1. Active router link text color (`!text-accent` via `active-class`)
2. Nav link hover state (`hover:text-accent`)
3. Primary CTA button background ("Explore Playground")
4. Theme dropdown trigger text on hover
5. Selected theme checkmark in dropdown
6. Focus ring on interactive elements

`--color-accent-cyan` is used ONLY on:

1. Tech stack category headings
2. Decorative accent in future phases (Skills Diamond Wall)

`--color-accent-yellow` is used ONLY on:

1. Decorative accent in future phases (terminal prompt, badges)

### Theme Transition

- Duration: 250ms
- Easing: ease-in-out
- Properties: `color`, `background-color`, `border-color` only (no `all`)
- Suppression on load: Add `no-transition` class to `:root` initially, remove after first paint via `requestAnimationFrame` to prevent color animation on page load
- Source: D-12 (locked decision)

### Component Details

#### ThemeDropdown

**Trigger:**

- Text label: "Theme" (14px, weight 400, `text-text-muted`)
- Down arrow: Unicode `\u25BE` (small down-pointing triangle) appended to label
- Hover: text changes to `text-accent`
- Position: right-justified in header, after nav links
- Click, Enter, Space, or ArrowDown opens the dropdown

**Dropdown Panel:**

- Background: `bg-surface-overlay`
- Border: 1px solid `border-border`
- Border radius: 8px
- Shadow: `0 4px 16px rgba(0,0,0,0.3)` (consistent dark shadow regardless of theme)
- Width: 220px fixed
- Max height: none (9 items always fit)
- Position: absolute, anchored below trigger, right-aligned
- Padding: 8px vertical (sm)

**Option Row:**

- Height: 36px (padded to comfortable click target)
- Padding: 8px horizontal (sm), 6px vertical
- Text: theme name in exact VSCode name (14px, weight 400, `text-text`)
- Checkmark: Unicode `\u2713` prefix (14px, `text-accent`) for currently confirmed theme
- Hover background: `bg-hover`
- Active/focused background: `bg-selection`
- Border radius on hover: 4px

**Keyboard Interaction (D-04, D-05, D-06):**

- ArrowDown/ArrowUp: Move focus through options, live-preview theme colors on `:root`
- Enter: Confirm selection, persist to localStorage, close dropdown
- Escape: Revert to previously confirmed theme, close dropdown
- Click-outside: Same as Escape (revert and close)
- Tab: Close dropdown (implicit confirm of current preview, or revert -- match Escape behavior)

**ARIA:**

- Trigger: `role="button"`, `aria-haspopup="listbox"`, `aria-expanded="true/false"`
- Panel: `role="listbox"`, `aria-label="Select theme"`
- Options: `role="option"`, `aria-selected="true/false"`, unique `id` per option
- Active tracking: `aria-activedescendant` on listbox points to focused option `id`

#### MobileMenu

**Trigger (hamburger):**

- Three horizontal lines (CSS-only, no SVG/icon library)
- Size: 44px x 44px touch target
- Position: right side of header, replaces theme dropdown on mobile
- Visible at: below `sm` breakpoint (640px)
- Lines: 3 bars, each 24px wide x 2px tall, `bg-text`, spaced 6px apart
- Animate to X when open: top/bottom bars rotate 45deg, middle bar fades

**Overlay:**

- Background: `bg-surface` with full viewport coverage (position: fixed, inset: 0)
- Z-index: 50 (above all content)
- Entry animation: fade in 200ms + slide from right 200ms (transform: translateX)
- Exit animation: reverse of entry
- Focus trap: `inert` attribute on `#app` content while menu is open

**Content layout:**

- Top: Close button (X) right-aligned, 44px touch target, `text-text`
- Nav section: vertical stack of nav links
  - Links: "Home", "CLI", "Playground" (24px, weight 700, `text-text`)
  - Active link: `text-accent`
  - Spacing: 24px (lg) between items
  - Each link is a full-width tap target (min 44px height)
  - Clicking a link navigates and closes menu
- Theme section: below nav, separated by 32px (xl) gap
  - Section label: "THEME" (14px, weight 700, uppercase, `text-text-muted`, letter-spacing 0.05em)
  - Theme list: vertical stack of theme names
  - Each option: 16px, weight 400, `text-text`, min 44px tap target height
  - Selected theme: checkmark prefix + `text-accent`
  - Selecting a theme applies it immediately (no preview/confirm flow on mobile)
  - Spacing: 8px (sm) between theme options

**Accessibility:**

- `aria-label="Main menu"` on overlay
- `aria-hidden="true"` on `#app` content when menu open
- Auto-focus close button on open
- Escape closes menu
- Return focus to hamburger trigger on close

#### TheHeader Modifications

**Desktop (>= sm / 640px):**

- Existing layout preserved: name left, nav links center-left
- Add: ThemeDropdown right-justified (pushed with `ml-auto`)
- Hamburger: hidden

**Mobile (< sm / 640px):**

- Name left
- Nav links: hidden (moved into MobileMenu)
- ThemeDropdown: hidden (moved into MobileMenu)
- Hamburger: visible, right-justified

## Copywriting Contract

## Interaction States

### Theme Dropdown

| State                | Visual                                                                       |
| -------------------- | ---------------------------------------------------------------------------- |
| Closed               | Trigger shows "Theme \u25BE" in `text-text-muted`                            |
| Hover on trigger     | Text changes to `text-accent`, cursor pointer                                |
| Open                 | Panel drops below trigger, current theme has checkmark, first option focused |
| Arrow browsing       | Focused option has `bg-selection` background, page colors live-preview       |
| Hover on option      | `bg-hover` background                                                        |
| Confirmed selection  | Checkmark moves to new theme, dropdown closes, localStorage updated          |
| Escape/click-outside | Colors revert to confirmed theme, dropdown closes                            |

### Mobile Menu

| State        | Visual                                       |
| ------------ | -------------------------------------------- |
| Closed       | Hamburger icon visible                       |
| Open         | Full-screen overlay, hamburger animates to X |
| Nav link tap | Navigate + close menu                        |
| Theme tap    | Apply theme immediately + checkmark moves    |
| Escape       | Close menu                                   |

### Theme Transition

| State                      | Visual                                                        |
| -------------------------- | ------------------------------------------------------------- |
| Page load (first visit)    | SynthWave '84 applied instantly (no transition)               |
| Page load (returning user) | Saved theme applied instantly via FOUC script (no transition) |
| Theme switch               | All colors transition 250ms ease-in-out                       |
| Arrow key preview          | Colors transition 250ms per arrow press                       |
| Escape revert              | Colors transition 250ms back to confirmed theme               |

## Theme Ordering (D-11 -- Claude's Discretion)

Dropdown order groups by type (dark first, then light), with default first:

1. SynthWave '84 (default)
2. Dark Modern
3. Dark+
4. Monokai Dimmed
5. Red
6. Solarized Dark
7. Dark High Contrast
8. Solarized Light
9. Light High Contrast

Rationale: Default first for easy revert. Dark themes grouped together (most users will stay dark). Light and high-contrast themes at the bottom as alternative options.
